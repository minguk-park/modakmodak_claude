# 커밋 생성

현재 변경사항을 확인하고 하나의 커밋을 생성한다.

## 프로세스
1. `git status`와 `git diff`로 변경사항 파악
2. 커밋 타입과 메시지 결정
3. 사용자에게 확인
4. 커밋 생성

## 커밋 타입

| 타입 | 설명 |
|------|------|
| feat | 새로운 기능 |
| fix | 버그 수정 |
| chore | 설정 변경, 라이브러리 등 잡일 |
| refactor | 동작 변경 없이 코드 정리 |
| style | 포맷팅, 세미콜론 등 |
| docs | 문서 작성/수정 |
| test | 테스트 코드 |
| ci | CI/CD 설정 |
| perf | 성능 개선 |

## 커밋 포맷
```
<type>: <description>

- 세부설명1
- 세부설명2
```

## 규칙
- 한글 커밋 메시지 사용
- 첫 글자 소문자
- 마침표 없이 끝내기
- 명령형으로 작성 (추가, 수정, 삭제)
- 모든 변경사항을 하나의 커밋으로 만든다
- 커밋 전에 메시지를 보여주고 확인받기
